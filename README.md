## Project Conventions

This repository follows the conventions defined in
[CONVENTIONS.md](CONVENTIONS.md), covering CI, linting, container, Makefile,
and documentation standards.

## Plan Documents

Every meaningful change includes a plan document in
[docs/plans/](docs/plans/). Plans record what was built, why, and what was
learned. See [docs/plans/TEMPLATE.md](docs/plans/TEMPLATE.md) for the
template.

## AI-Orchestrated SDLC Framework

This repository's conventions are informed by the
[AI-Orchestrated Software Development Lifecycle](https://gist.github.com/clcollins/79444e363dad4dfa8053f3b2eecc0e9d)
framework. The full framework covers multi-agent orchestration across design,
review, integration, and production validation. This repo adopts the
standalone principles: plan documents, Lessons Learned, create-only posture,
and context isolation.
